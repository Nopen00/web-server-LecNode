# 학교 출석 프로그램 개발 견적서

## 📋 프로젝트 개요

**프로젝트명**: 학교 출석 프로그램 (School Attendance Program)  
**제출 기한**: 2024년 12월 16일  
**개발 목표**: 2024년 11월 중 전반적인 기술적 부분 완성 (최적화/디자인 제외)  
**개발 범위**: 백엔드, 프론트엔드, 데이터베이스 전체 개발  
**팀 구성**: 2명 (작업 분배는 견적서 기반으로 결정)  
**유지보수**: 기말 시험 후 유지보수 예정 없음 (코드 재사용 목적)  
**예산**: 무료 도구/서비스 우선 사용, 필수 유료 서비스는 필요 시 결정

---

## 🛠 기술 스택

### 백엔드
- **언어**: Python (확정)
- **프레임워크**: Django (최종 확정)
- **추가 라이브러리**: 
  - 인증: JWT 토큰 관리
  - 파일 업로드: 멀티파트 파일 처리
  - 날짜/시간: 공휴일 처리, 반복 규칙 처리

### 프론트엔드
- **프레임워크**: React
- **상태 관리**: Context API (결정됨)
- **UI 라이브러리**: Chakra UI (결정됨)
- **HTTP 클라이언트**: Axios
- **라우팅**: React Router

### 데이터베이스
- **DBMS**: MySQL
- **ORM**: Django ORM

### 기타
- **파일 저장소**: 로컬 저장소 (확정)
- **알림 시스템**: 폴링 방식 (확정)

## 📊 데이터베이스 스키마 설계

### 주요 테이블 (예상)

#### 1. User (사용자)
- `id` (PK)
- `role` (ENUM: admin, instructor, student)
- `name`
- `email`
- `password_hash`
- `student_id` (학번, 학생인 경우)
- `department_id` (FK)
- `created_at`, `updated_at`

**결정 사항**: 
- 학번 형식: 202321063 (입학년도4+학과2+순서3)
- 이메일: 중복 불허용
- 비밀번호: 최소 6자, 대소문자+숫자+특수문자 필수

#### 2. Department (학과)
- `id` (PK)
- `name` (학과명)
- `code` (학과 코드)
- `created_at`, `updated_at`

**결정 사항**: 예시대로 (CS, EE 등)

#### 3. Semester (학기)
- `id` (PK)
- `year` (연도)
- `term` (학기: 1, 2, 여름, 겨울)
- `start_date`
- `end_date`
- `created_at`, `updated_at`

**결정 사항**: 예시대로

#### 4. Course (과목)
- `id` (PK)
- `title` (과목명)
- `code` (과목 코드)
- `section` (분반)
- `instructor_id` (FK → User)
- `semester_id` (FK → Semester)
- `department_id` (FK → Department)
- `room` (강의실)
- `created_at`, `updated_at`

**결정 사항**: 
- 과목 코드: 예시대로 (CS101)
- 분반: 최대 4분반

#### 5. ClassSession (수업 세션)
- `id` (PK)
- `course_id` (FK → Course)
- `week` (주차)
- `session_number` (회차)
- `start_at` (시작 시간)
- `end_at` (종료 시간)
- `room` (강의실)
- `attendance_method` (ENUM: electronic, code, roll_call)
- `attendance_code` (인증번호, nullable)
- `status` (ENUM: scheduled, open, paused, closed)
- `is_holiday` (공휴일 여부)
- `is_makeup` (보강일 여부)
- `created_at`, `updated_at`

**결정 사항**:
- 인증번호: 6자리 숫자, 자동 생성
- 호명: 교원이 직접 체크 가능
- 출석 체크 시간: 교수가 출석 시작 후 3~15분 설정 가능
- 공휴일: 네이버 달력 기준 (수동 입력)

#### 6. Enrollment (수강신청)
- `id` (PK)
- `user_id` (FK → User)
- `course_id` (FK → Course)
- `role` (ENUM: student, ta)
- `enrolled_at`
- `created_at`, `updated_at`

**결정 사항**:
- 엑셀/CSV 형식: 개발자가 결정
- 수강 취소: 필요

#### 7. Attendance (출석)
- `id` (PK)
- `session_id` (FK → ClassSession)
- `student_id` (FK → User)
- `status` (INT: 0=미정, 1=출석, 2=지각, 3=결석, 4=공결)
- `checked_at` (출석 체크 시간)
- `late_minutes` (지각 시간, 분 단위)
- `location` (전자출결 시 위치 정보, nullable)
- `created_at`, `updated_at`

**결정 사항**:
- 지각 기준: 수업 시작 후 10분
- 지각→결석: 30분 이상

#### 8. ExcuseRequest (공결 신청)
- `id` (PK)
- `session_id` (FK → ClassSession)
- `student_id` (FK → User)
- `reason_code` (사유 코드)
- `reason_text` (상세 사유)
- `status` (ENUM: pending, approved, rejected)
- `instructor_comment` (교원 코멘트, nullable)
- `files` (JSON: 파일 경로 배열)
- `submitted_at`
- `reviewed_at`
- `created_at`, `updated_at`

**결정 사항**:
- 사유 코드: 병가, 경조사, 기타
- 파일: 최대 1개, 사진/PDF/오피스/한글/ZIP
- 신청 기한: 하루 전 기본, 당일 이후는 교수 선택 가능

#### 9. Appeal (이의제기)
- `id` (PK)
- `attendance_id` (FK → Attendance)
- `student_id` (FK → User)
- `message` (이의 내용)
- `status` (ENUM: pending, approved, rejected)
- `instructor_comment` (교원 코멘트, nullable)
- `submitted_at`
- `reviewed_at`
- `created_at`, `updated_at`

**결정 사항**: 학기 종료 후 다음주 금요일까지

#### 10. Announcement (공지사항)
- `id` (PK)
- `course_id` (FK → Course)
- `instructor_id` (FK → User)
- `title`
- `content`
- `created_at`, `updated_at`

**결정 사항**: 작성자와 관리자 모두 수정/삭제 가능, 고정 기능 선택 가능

#### 11. Message (메시지)
- `id` (PK)
- `course_id` (FK → Course)
- `sender_id` (FK → User)
- `receiver_id` (FK → User)
- `subject`
- `content`
- `is_read` (읽음 여부)
- `created_at`, `updated_at`

**결정 사항**: 보낸 사람만 삭제 가능

#### 12. Poll (투표)
- `id` (PK)
- `course_id` (FK → Course)
- `instructor_id` (FK → User)
- `title` (투표 제목)
- `description` (설명)
- `session_id` (FK → ClassSession, 공강 대상 수업)
- `deadline` (마감 시간)
- `status` (ENUM: open, closed)
- `created_at`, `updated_at`

#### 13. PollVote (투표 참여)
- `id` (PK)
- `poll_id` (FK → Poll)
- `student_id` (FK → User)
- `vote` (ENUM: agree, disagree)
- `created_at`

**결정 사항**: 투표 생성 시 선택 가능 (실시간/마감 후, 재투표 가능 여부)

#### 14. AttendancePolicy (출석 정책)
- `id` (PK)
- `course_id` (FK → Course)
- `late_threshold` (지각 기준, 분)
- `late_to_absent_threshold` (지각→결석 전환 기준, 분)
- `max_absences` (최대 결석 허용 횟수)
- `absence_warning_threshold` (경고 결석 횟수, 예: 2회)
- `absence_danger_threshold` (위험 결석 횟수, 예: 3회)
- `created_at`, `updated_at`

**결정 사항**: 과목별 정책, 결석 25% 이상 시 낙제, 출석=1 지각=0.5

#### 15. AuditLog (감사 로그)
- `id` (PK)
- `user_id` (FK → User, 수행한 사용자)
- `action` (작업 유형: attendance_change, approval, policy_change 등)
- `target_type` (대상 타입: Attendance, ExcuseRequest 등)
- `target_id` (대상 ID)
- `old_value` (JSON, 변경 전 값)
- `new_value` (JSON, 변경 후 값)
- `ip_address`
- `created_at`

**결정 사항**: 로그 보관 1년, 관리자만 조회

#### 16. File (파일)
- `id` (PK)
- `original_name` (원본 파일명)
- `stored_path` (저장 경로)
- `mime_type`
- `size` (바이트)
- `uploader_id` (FK → User)
- `related_type` (관련 타입: excuse_request, appeal 등)
- `related_id` (관련 ID)
- `created_at`

## 🔌 API 엔드포인트 설계

### 인증 (Authentication)
- `POST /api/auth/login` - 로그인
- `POST /api/auth/refresh` - 토큰 갱신
- `POST /api/auth/logout` - 로그아웃

**결정 사항**: 개발자가 결정

### 사용자 관리 (User Management)
- `GET /api/users` - 사용자 목록 (관리자)
- `GET /api/users/{id}` - 사용자 상세
- `POST /api/users` - 사용자 등록 (관리자)
- `PUT /api/users/{id}` - 사용자 수정 (관리자)
- `DELETE /api/users/{id}` - 사용자 삭제 (관리자)
- `PUT /api/users/{id}/role` - 권한 변경 (관리자)

**결정 사항**: 하드 삭제

### 학과/학기/과목 관리 (Department/Semester/Course)
- `GET /api/departments` - 학과 목록
- `POST /api/departments` - 학과 생성 (관리자)
- `PUT /api/departments/{id}` - 학과 수정 (관리자)
- `DELETE /api/departments/{id}` - 학과 삭제 (관리자)

- `GET /api/semesters` - 학기 목록
- `POST /api/semesters` - 학기 생성 (관리자)
- `PUT /api/semesters/{id}` - 학기 수정 (관리자)
- `DELETE /api/semesters/{id}` - 학기 삭제 (관리자)

- `GET /api/courses` - 과목 목록
- `GET /api/courses/{id}` - 과목 상세
- `POST /api/courses` - 과목 생성 (관리자)
- `PUT /api/courses/{id}` - 과목 수정 (관리자)
- `DELETE /api/courses/{id}` - 과목 삭제 (관리자)

**결정 사항**: 하드 삭제

### 수업 세션 관리 (Class Session)
- `GET /api/courses/{id}/sessions` - 수업 세션 목록
- `POST /api/courses/{id}/sessions` - 수업 세션 생성 (교원)
- `PUT /api/sessions/{id}` - 수업 세션 수정 (교원)
- `DELETE /api/sessions/{id}` - 수업 세션 삭제 (교원)
- `POST /api/sessions/{id}/open` - 출석 열기 (교원)
- `POST /api/sessions/{id}/pause` - 출석 일시정지 (교원)
- `POST /api/sessions/{id}/close` - 출석 마감 (교원)
- `GET /api/sessions/{id}/attendance-code` - 인증번호 조회/생성 (교원)

**결정 사항**: 반복 규칙 기반 생성, 공휴일 수동 입력, 보강일 수동 추가

### 출석 관리 (Attendance)
- `POST /api/sessions/{id}/attend` - 출석 체크 (학생)
- `GET /api/sessions/{id}/attendance` - 출석 현황 조회
- `GET /api/sessions/{id}/attendance/summary` - 출석 요약 (대시보드)
- `PATCH /api/attendance/{id}` - 출석 정정 (교원)
- `GET /api/courses/{id}/attendance` - 과목별 출석 현황

**결정 사항**: 교수가 출석 시작 후 3~15분 설정 가능

### 공결 관리 (Excuse Request)
- `POST /api/sessions/{id}/excuses` - 공결 신청 (학생)
- `GET /api/excuses` - 공결 신청 목록 (교원: `?status=pending`)
- `GET /api/excuses/{id}` - 공결 신청 상세
- `PATCH /api/excuses/{id}` - 공결 승인/반려 (교원)

**결정 사항**: 병가, 경조사, 기타

### 이의제기 (Appeal)
- `POST /api/attendance/{id}/appeals` - 이의제기 신청 (학생)
- `GET /api/appeals` - 이의제기 목록 (교원: `?status=pending`)
- `GET /api/appeals/{id}` - 이의제기 상세
- `PATCH /api/appeals/{id}` - 이의제기 처리 (교원)

### 수강신청 (Enrollment)
- `GET /api/courses/{id}/enrollments` - 수강생 목록
- `POST /api/courses/{id}/enrollments` - 수강신청 (또는 관리자 등록)
- `DELETE /api/enrollments/{id}` - 수강 취소
- `POST /api/enrollments/import` - 엑셀/CSV 일괄 등록 (관리자)

**결정 사항**: 개발자가 결정

### 공지사항 (Announcement)
- `GET /api/courses/{id}/announcements` - 공지사항 목록
- `GET /api/announcements/{id}` - 공지사항 상세
- `POST /api/courses/{id}/announcements` - 공지사항 작성 (교원)
- `PUT /api/announcements/{id}` - 공지사항 수정 (교원)
- `DELETE /api/announcements/{id}` - 공지사항 삭제 (교원)

### 메시지 (Message)
- `GET /api/messages` - 메시지 목록 (받은/보낸)
- `GET /api/messages/{id}` - 메시지 상세
- `POST /api/messages` - 메시지 전송
- `PATCH /api/messages/{id}/read` - 읽음 처리
- `DELETE /api/messages/{id}` - 메시지 삭제

**결정 사항**: 개발자가 결정 (폴링 권장)

### 투표 (Poll)
- `GET /api/courses/{id}/polls` - 투표 목록
- `GET /api/polls/{id}` - 투표 상세
- `POST /api/courses/{id}/polls` - 투표 생성 (교원)
- `POST /api/polls/{id}/vote` - 투표 참여 (학생)
- `GET /api/polls/{id}/results` - 투표 결과 조회
- `POST /api/polls/{id}/close` - 투표 마감 (교원)

### 정책/점수 (Policy/Score)
- `GET /api/courses/{id}/policy` - 출석 정책 조회
- `PUT /api/courses/{id}/policy` - 출석 정책 설정 (교원)
- `GET /api/courses/{id}/score/attendance` - 출석 점수 조회

**결정 사항**: 가중치 적용 (출석=1, 지각=0.5)

### 리포트/내보내기 (Report/Export)
- `GET /api/reports/attendance` - 출석 리포트 (`?course_id=...&week=...`)
- `GET /api/reports/attendance/export` - 출석 리포트 엑셀 다운로드
- `GET /api/reports/risk-students` - 위험군 학생 목록
- `GET /api/reports/system-status` - 시스템 상태 리포트 (관리자)

**결정 사항**: 엑셀 형식, 위험군 기준 15% 이상 결석

### 파일 업로드 (File Upload)
- `POST /api/files` - 파일 업로드 (증빙 파일 등)
- `GET /api/files/{id}` - 파일 다운로드
- `DELETE /api/files/{id}` - 파일 삭제

**결정 사항**: 로컬 저장소, 과제 파일은 업로더/교원/관리자만, 게시판 파일은 수강생 모두

### 감사 로그 (Audit Log)
- `GET /api/audits` - 감사 로그 조회 (관리자)
  - 쿼리 파라미터: `?target_type=Attendance&target_id=...&user_id=...&start_date=...&end_date=...`

### 대시보드 (Dashboard)
- `GET /api/dashboard` - 대시보드 데이터 (역할별로 다른 데이터 반환)
  - 학생: 내 출석 현황, 공결 신청 상태, 알림
  - 교원: 담당 수업 출석 현황, 승인 대기 건수, 알림
  - 관리자: 시스템 통계, 최근 활동

## 🎨 프론트엔드 페이지/컴포넌트 설계

### 인증 관련
- `/login` - 로그인 페이지
- `/register` - 회원가입 페이지 (관리자만?)

### 대시보드
- `/dashboard` - 역할별 대시보드
  - 학생: 출석 현황 요약, 최근 알림, 공결 신청 상태
  - 교원: 담당 수업 목록, 승인 대기 건수, 최근 알림
  - 관리자: 시스템 통계, 최근 활동

### 관리자 페이지
- `/admin/departments` - 학과 관리
- `/admin/semesters` - 학기 관리
- `/admin/courses` - 과목 관리
- `/admin/users` - 사용자 관리
- `/admin/audit-logs` - 감사 로그 조회
- `/admin/reports` - 리포트 생성

### 교원 페이지
- `/instructor/courses` - 담당 과목 목록
